# Claude Code Evolution - Simple AI-Driven Algorithm Improvement

This document describes how to use Claude to automatically improve fish behavior algorithms through iterative evolution cycles.

## Quick Start

Simply say to Claude:

```
"Run AI evolution"
"Evolve the code pool"
"Improve the worst algorithm"
```

That's it! Claude will handle the entire process automatically.

## What Claude Does Automatically

When you request AI evolution, Claude performs these steps:

### 1. Run Simulation & Collect Data
```bash
python main.py --headless --max-frames 10000 --export-stats evolution_results.json
```
- Runs a headless simulation (10,000 frames by default)
- Collects performance statistics for all 53 algorithms
- Exports comprehensive JSON with reproduction rates, survival rates, death causes, etc.

### 2. Analyze & Identify Worst Performer
Claude examines the performance data and identifies:
- **Worst performer**: Algorithm with lowest reproduction rate (needs ≥5 births for statistical significance)
- **Root causes**: Primary death causes (starvation, predation, old age)
- **Specific problems**: Why the algorithm is failing

Example findings:
```
CircularHunter:
- Reproduction rate: 0.0% (TERRIBLE)
- Avg lifespan: 390 frames (only 13 seconds!)
- Deaths: 100% from starvation
- Food eaten: Only 3 total
- Critical bug: Returns (0,0) when no food visible - fish stops moving!
```

### 3. Read Source Code & Generate Improvements
Claude:
- Reads the algorithm's source file using the registry mapping
- Analyzes the `execute()` method logic
- Identifies critical bugs and inefficiencies
- Generates improved code with:
  - Better food-seeking behavior
  - Energy-aware decision making
  - Smarter predator avoidance
  - Active exploration instead of stopping
  - Faster/more efficient movement patterns

### 4. Commit & Push Changes
```bash
git add -A
git commit -m "AI Evolution: Improve [AlgorithmName]

Previous performance (POOR):
- Reproduction rate: X%
- Main issues: [specific problems]

Root causes:
1. [Bug/issue description]
2. [Bug/issue description]

Improvements made:
1. [Specific fix]
2. [Specific fix]

Expected outcome:
- Better survival rate
- Higher reproduction
- More efficient behavior

Generated by: Claude (AI Code Evolution)"

git push -u origin claude/run-ai-evolution-[SESSION_ID]
```

### 5. Validate with Longer Simulation (Optional)
If requested, Claude runs an extended validation:
```bash
python main.py --headless --max-frames 50000 --export-stats evolution_results_long.json
```
- 5x longer simulation for better statistical data
- Verifies improvements propagate through generations
- Shows which algorithms dominate through natural selection

## Algorithm Selection Criteria

Claude identifies the worst performer using these rules:

1. **Minimum sample size**: Algorithm must have ≥5 births for statistical significance
2. **Primary metric**: Lowest reproduction rate
3. **Secondary factors**:
   - High starvation rate (food-seeking problems)
   - Short average lifespan
   - Low energy efficiency
   - Zero or negative population growth

If multiple algorithms tie, Claude prioritizes those with clear, fixable problems.

## Types of Improvements Claude Makes

### 1. Critical Bug Fixes
- **Stopped movement**: Algorithms that return (0,0) when no food visible
- **Infinite loops**: Getting stuck in circling or patrol patterns
- **Division by zero**: Crashes from improper distance calculations

### 2. Energy-Aware Behavior
```python
# BEFORE: Always circle food
if distance < strike_threshold:
    strike()
else:
    circle_around_food()

# AFTER: Skip inefficient behavior when hungry
energy_ratio = fish.energy / fish.max_energy
if energy_ratio < 0.5:  # Hungry - go direct!
    direction = normalize(food.pos - fish.pos)
    return direction * 1.3
elif distance < strike_threshold:
    strike()
else:
    circle_around_food()
```

### 3. Active Exploration
```python
# BEFORE: Stop when no food visible
if not nearest_food:
    return 0, 0  # TERRIBLE!

# AFTER: Actively explore to find food
if not nearest_food:
    self.exploration_direction += random.uniform(-0.3, 0.3)
    exploration_vec = Vector2(
        math.cos(self.exploration_direction),
        math.sin(self.exploration_direction)
    )
    return exploration_vec * exploration_speed
```

### 4. Optimized Parameters
- **Faster movement**: Speed multipliers increased (0.8 → 1.2)
- **Larger detection ranges**: Strike distances widened (30 → 80 pixels)
- **Better thresholds**: Energy thresholds tuned (0.2 → 0.3)
- **Smarter speeds**: Dynamic speeds based on urgency

### 5. Smarter Predator Avoidance
```python
# Energy-aware flee distance
flee_distance = 80 if is_desperate else 110

# Conserve energy when fleeing
flee_speed = 1.2 if is_desperate else 1.4
```

## Tips for Best Results

### 1. Run Long Simulations
- Short runs (1k-5k frames): Too random, not enough data
- Medium runs (10k-20k frames): Good for quick iteration
- Long runs (50k-100k frames): Best for validation

### 2. Let Natural Selection Work
Don't expect every improvement to dominate immediately. Good algorithms will spread through the population over multiple generations.

### 3. Iterate Multiple Times
The best results come from improving 3-5 algorithms in succession. Each improvement makes the overall population stronger.

### 4. Focus on Root Causes
Claude identifies the specific bugs causing failures:
- Starvation → Food-seeking problems
- Predation → Avoidance too weak
- Energy efficiency → Movement too expensive

### 5. Trust the Data
Reproduction rate is the ultimate metric. If an algorithm reproduces, it's working!

## Architecture Notes

### Registry System
`core/registry.py` maps algorithm names to source files:
```python
{
  "circular_hunter": {
    "source_file": "/home/user/tank/core/algorithms/food_seeking.py",
    "class_name": "CircularHunter",
    "category": "food_seeking"
  }
}
```

This allows Claude to:
- Find the exact file to edit
- Target specific algorithms
- Make surgical code changes

### Performance Tracking
`core/enhanced_statistics.py` tracks comprehensive metrics:
- Birth/death counts per algorithm
- Reproduction rates (offspring per parent)
- Survival rates (still alive / total born)
- Energy efficiency (food eaten per fish)
- Death cause breakdown
- Lifespan distributions
